## 2. 실행 가능성

### 비현실적 일정 구간
- **2주차**: T05~T13-c를 1주일에 완료해야 함. step2(비동기 API 호출 + 폴백 로직 + XML 파싱)만 해도 디버깅 포함 최소 1~2일 소요 예상. 여기에 step3(OpenAI 연동) + 전체 파이프라인 실행(6~8시간)까지 포함되어 가장 밀도 높은 주차임.
- **T13-c**: 전체 파이프라인 실행이 6~8시간이지만, 중간에 실패할 경우 원인 분석 + 재실행 시간 미반영. 실패 1회당 +3~4시간 추가.

### 미확보 역량/자원
- Python 비동기(asyncio + aiohttp) + XML 파싱 + Supabase SDK 경험이 전제됨 — 학습 곡선 미반영.
- Next.js 15 App Router + generateStaticParams + ISR 경험이 전제됨.

### 병목 의존성
- **B550928 API 키 승인** (1~2일): T10부터 블로킹. 계획에서 인지하고 있으나 실제 승인이 3일+ 걸릴 경우 2주차 전체가 밀림.
- **Supabase Storage 업로드 (T02)**: xlsx 파일을 수동으로 확보해야 하므로, 공공데이터포털 다운로드 지연 시 step1 테스트 불가.

## 4. 최악의 시나리오 3가지

### 시나리오 1: 구글 AI 생성 콘텐츠 페널티
- **상황**: 2만 페이지의 AI 생성 SEO 글이 구글 Helpful Content Update 또는 Spam 정책에 의해 색인 거부 또는 순위 하락
- **발생 확률**: **중간** — 구글은 2024년부터 대량 AI 콘텐츠 사이트에 대한 제재를 강화하고 있음
- **피해 규모**: **치명적** — 프로젝트의 핵심 수익 모델(검색 트래픽 → 애드센스)이 완전히 무력화됨. 4주간의 개발 노력이 수포로 돌아감

### 시나리오 2: B550928 API 응답 구조 변경 또는 서비스 중단
- **상황**: 공공데이터 API가 사전 통보 없이 응답 구조를 변경하거나, 일시 중단됨. step2 전체 데이터 수집 실패.
- **발생 확률**: **중간** — 공공데이터 API는 안정성이 낮은 것으로 알려져 있음
- **피해 규모**: **높음** — 상세 데이터 없이는 상세 페이지가 빈 껍데기가 되어 SEO 가치 상실

### 시나리오 3: Vercel 무료 티어 빌드 제한 초과
- **상황**: 2만 페이지 SSG 빌드가 Vercel Hobby 플랜의 빌드 시간 제한(45분) 또는 함수 실행 시간 제한을 초과
- **발생 확률**: **높음** — 2만 페이지 generateStaticParams는 빌드 시간이 상당히 길 수 있음
- **피해 규모**: **높음** — 배포 전략을 ISR/On-demand로 전환해야 하며, 아키텍처 변경에 추가 시간 소요

## 5. 논리적 일관성

| # | 불일치 항목 | 설명 |
|---|---|---|
| C1 | **DT01 "확정" vs 체크리스트 "미완"** | 본문 DT01~DT04는 "확정"으로 표기되었으나, 7장 체크리스트에서는 모두 `- [ ]` 미완료 상태. 동기화 필요 |

## 6. 판정: 🟡 조건부 승인

### 종합 근거

계획서는 전체 구조가 체계적이고, 태스크 분해·선행 조건·리스크 관리까지 포함한 높은 완성도를 보인다. 그러나 아래 항목들이 해결되기 전까지 실행 보류를 권고한다:

**반드시 해결해야 할 항목 (실행 전)**:
1. **구글 AI 콘텐츠 정책 리스크 대응**: 2만 건 AI 생성 콘텐츠의 색인 거부 가능성에 대한 대안 필요. 소규모(100건) 파일럿 배포 → 색인 확인 → 전체 확장 단계를 추가할 것
2. **Vercel 2만 페이지 빌드 가능성 검증**: Hobby 플랜에서 generateStaticParams 2만 건 빌드가 45분 제한 내에 가능한지 사전 검증. 불가 시 ISR fallback 전략 수립
3. **개인정보보호법/공공데이터 이용 조건 확인**: 공공데이터의 상업적 2차 가공·배포에 대한 법적 근거 확인

**권고 개선 사항 (실행 중 반영 가능)**:
4. 체크리스트와 의사결정 상태 동기화 (C1)
5. T05, T15의 불필요한 선행 조건 의존성 제거 (C2, C5)
6. Vercel KV 무료 한도와 비용 $0 목표의 정합성 확인 (C3, N6)
7. 데이터 정합성 검증 쿼리 추가 (N5)
8. 에러 복구/부분 재실행 전략 명문화 (N1)
