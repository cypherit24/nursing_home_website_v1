- 공공데이터 API 구조 및 매핑 정보

# 완료 기준

plan.md가 아래 조건을 모두 만족하면 완료로 간주한다:

1. 각 태스크는 Claude Code 프롬프트 1회 실행으로 완결 가능한 단위로 분리되어 있다
2. 각 태스크에 선행 조건(dependency)이 명시되어 있다
3. 각 태스크에 완료 확인 방법(검증 기준)이 구체적으로 기술되어 있다
4. 막힌 점과 합의 필요 항목이 태스크 흐름에 반영되어 있다
5. 전체 일정이 주차별로 구성되어 있다
6. 비용이 발생하는 태스크는 예상 비용이 명시되어 있다
7. 각 태스크의 실행 환경(Codespaces / Supabase 웹 / GitHub Actions / 브라우저)이 명시되어 있다

# plan.md 구조 지시

아래 형식을 정확히 따라 plan.md를 작성해줘.

## 1. 프로젝트 개요
- 한 줄 목표 문장
- 핵심 성공 지표 3가지 (KPI)
- 전체 예상 기간

## 2. 의사결정 선행 항목
코딩 착수 전에 반드시 결정해야 할 사항들을 표 형태로 정리.
각 항목에: 결정 내용 / 옵션 / 추천안 / 결정 기한 포함.

## 3. 마일스톤
전체 작업을 4~5개의 마일스톤으로 묶어 표로 정리.
각 마일스톤에: 번호 / 이름 / 포함 태스크 범위 / 완료 기준 / 예상 기간 포함.

## 4. 태스크 목록
각 태스크를 아래 형식으로 작성:

### T[번호]: [태스크 이름]
- 마일스톤: M[번호]
- 실행 환경: [Codespaces / Supabase 웹 / GitHub Actions / 브라우저]
- 선행 조건: [선행 태스크 번호 또는 "없음"]
- 작업 내용: 3줄 이내 요약
- 완료 확인: 구체적인 검증 방법 (어디서 무엇을 확인하는지)
- 예상 소요: [시간]
- 비용: [없음 / 예상 금액]

## 5. 리스크 관리
발생 가능한 리스크를 표로 정리.
각 항목에: 리스크 / 발생 확률 / 영향도 / 대응 방안 포함.

## 6. 일정표 (주차별)
1주차부터 완료 예상 주차까지 간트 형태 텍스트로 정리.
각 주차에 완료 예정 태스크 번호 포함.

## 7. 체크리스트
전체 태스크를 [ ] 체크박스 형태로 나열.
완료 시 [x]로 변경할 수 있는 형태로 작성.

# 추가 지시사항

- 태스크 번호는 T01부터 순서대로 부여한다
- 마일스톤 번호는 M1부터 순서대로 부여한다
- TEST_MODE=true 소규모 검증 태스크와 전체 실행 태스크를 반드시 분리한다
- step2(API 수집)와 step3(SEO 생성) 태스크는
  "테스트 5건 검증" → "전체 실행 승인" → "전체 실행" 3단계로 나눠라
- 비용이 발생하는 태스크(OpenAI API)는 반드시 이전 단계 검증 완료 후에만 실행하도록
  선행 조건에 명시한다
- handoff.md의 "합의가 필요한 점" 5개 항목은
  각각 별도 의사결정 태스크(DT01~DT05)로 분리해서 태스크 목록 앞에 배치한다
- 평가등급 미확보, 사진 미확보 이슈는 리스크 항목에 포함한다
- plan.md는 마크다운 형식으로 출력하고
  코드 블록 없이 바로 복사해서 .md 파일로 저장할 수 있게 한다
